検索機能の実装

やること: 書籍一覧画面に、キーワード（タイトル・著者・出版社のOR部分一致）／ジャンル／ステータスでの絞り込み機能を追加。

1. 設計方針

  • UI：キーワード入力欄1つ＋ジャンル・ステータスのセレクトを並べるシンプル構成
  • HTTPメソッド：GET（URLに条件が載るのでブックマーク・共有可能）
  • 部分一致：キーワードはタイトル・著者・出版社のいずれかに含まれればヒット（OR）
  • 検索後の入力値保持：サーバーから $filters で入力値を返してフォームに再表示

2. コード

2-1. BookController@index の書き換え

app/Http/Controllers/BookController.php

public function index(Request $request)
{
    $query = Book::with('genre');

    // キーワード（タイトル・著者・出版社のOR部分一致）
    if ($request->filled('keyword')) {
        $keyword = $request->keyword;
        $query->where(function ($q) use ($keyword) {
            $q->where('title', 'like', "%{$keyword}%")
              ->orWhere('author', 'like', "%{$keyword}%")
              ->orWhere('publisher', 'like', "%{$keyword}%");
        });
    }

    // ジャンル
    if ($request->filled('genre_id')) {
        $query->where('genre_id', $request->genre_id);
    }

    // ステータス
    if ($request->filled('status')) {
        $query->where('status', $request->status);
    }

    $books = $query->orderBy('created_at', 'desc')->get();

    return view('books.index', [
        'books'    => $books,
        'genres'   => Genre::orderBy('id')->get(),
        'statuses' => BookStatus::cases(),
        'filters'  => $request->only(['keyword', 'genre_id', 'status']),
    ]);
}

3. ポイント解説

3-1. filled() を使う理由

$request->filled('keyword') は「値が存在し、かつ空文字ではない」を判定する。isset や has() だと空文字も true になってしまい、不要な WHERE が付く。「すべて」を選んだ場合は value="" が送られるので、filled() で自動的に除外できる。

3-2. OR条件をクロージャでグルーピングする

キーワードのOR条件は where(function ($q) use ($keyword) { ... }) で囲む必要がある。囲まないとSQLの優先順位が崩れる：

-- 囲まないとこうなる（意図しない）
WHERE title LIKE '%x%' OR author LIKE '%x%' OR publisher LIKE '%x%' AND genre_id = 3

-- 囲うとこうなる（意図どおり）
WHERE (title LIKE '%x%' OR author LIKE '%x%' OR publisher LIKE '%x%') AND genre_id = 3

AND は OR より優先度が高いため、ジャンル条件だけに AND が効き、キーワード条件のほとんどがOR扱いで広く拾われてしまう。

3-3. GETメソッドを選ぶ理由

  • 検索は「リソースの取得」＝冪等な操作なのでGETが適切
  • URLにクエリが載るので、ブックマーク・共有・リロードが有効
  • ブラウザの戻るボタンで前の検索結果に戻れる

3-4. $filters でフォーム入力を保持

$request->only([...]) で必要なキーだけ抜き出し、ビューに渡す。これにより検索後も入力欄や選択状態が残る。

4. 発生したエラー・原因・対応

4-1. ステータス検索でURLに status が載らない

症状: ステータスを選んで検索しても全件表示される。URLは ?keyword=&genre_id= でステータスのパラメータがない。プルダウンも「すべて」にリセットされる。
原因: 検索フォームの <select> に name="status" 属性が抜けていた。

対応: name="status" を追加。HTMLフォームは name 属性がない要素は送信対象外になる。

4-2. キーワード検索で一切ヒットしない

症状: タイトルの一部を入力しても、完全一致でも検索結果0件。URLには keyword=サンクチュアリ と正しく載っている。
原因: PHPのダブルクォート内で変数展開の $ が抜けていた："%{keyword}%"

対応: "%{$keyword}%" に修正。{$変数名} が変数展開の正しい書き方。{keyword} は単なる文字列として扱われ、SQL上は「{keyword}」という文字列を含む本を探すことになっていた。

補足: シンプルに "%$keyword%" でも動作するが、{$keyword} のほうが変数の境界が明確で安全。

5. 動作確認

  1. キーワードに本のタイトルの一部を入力 → 絞り込まれる
  2. ジャンルだけ選択して検索 → そのジャンルの本のみ
  3. 複数条件を組み合わせ → AND で絞り込み
  4. 「クリア」で初期状態に戻る
  5. URLが ?keyword=xxx&genre_id=3&status=reading のようになる
