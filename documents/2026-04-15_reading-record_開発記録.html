1. 今日の作業手順

1-1. Sail 起動時のエラー対応

エラー:
Bind for 0.0.0.0:80 failed: port is already allocated

原因: 別プロジェクト（todo-app）のSailコンテナが既にポート80を使用していた。

対応:

  • docker ps と lsof -iTCP:80 で占有プロセスを特定
  • todo-app 側を sail down で停止
  • reading-record を sail up -d で起動成功
別解: .env に APP_PORT=8080 を追加すれば、ポートをずらして両方起動も可能。

1-2. セッションテーブル未作成エラー

エラー:
Base table or view not found: Table 'laravel.sessions' doesn't exist

対応: マイグレーションが未実行だったため実行予定（本日は別作業を優先し、後続のマイグレーションと一緒に実行する方針）。

1-5. マイグレーション作成

genres テーブル:

Schema::create('genres', function (Blueprint $table) {
    $table->id();
    $table->string('name')->unique();
    $table->timestamps();
});

books テーブル:

Schema::create('books', function (Blueprint $table) {
    $table->id();
    $table->string('title');
    $table->string('author');
    $table->string('publisher')->nullable();
    $table->foreignId('genre_id')->constrained()->restrictOnDelete();
    $table->date('started_at')->nullable();
    $table->date('finished_at')->nullable();
    $table->string('status')->default('unread');
    $table->text('summary')->nullable();
    $table->text('impression')->nullable();
    $table->timestamps();
    $table->softDeletes();
});

1-6. PHP Enum 作成

app/Enums/BookStatus.php

enum BookStatus: string
{
    case Unread = 'unread';
    case Reading = 'reading';
    case Read = 'read';

    public function label(): string
    {
        return match ($this) {
            self::Unread => '未読',
            self::Reading => '読書中',
            self::Read => '読了',
        };
    }
}

2. 学びと判断ポイント

2-1. DB enum か PHP Enum か

DBのenum型DB string + PHP Enum
値の制約DB側で強制アプリ側（Enumクラス）で制御
選択肢変更コスト高い（ALTER TABLE 必要）低い（コード修正のみ）
型安全性PHP側は文字列のままIDE補完・比較が型で効く
DB移行耐性DB依存非依存

採用: DB は string、PHP は Enum クラスでキャスト。学習効果が高くメンテナンスも楽。

2-2. 外部キー制約の挙動

  • cascadeOnUpdate(): 親の id 変更時に子も追随（ほぼ使わない）
  • cascadeOnDelete(): 親削除で子も削除
  • restrictOnDelete(): 子がいれば親削除を禁止（今回採用）
  • nullOnDelete(): 親削除で子の外部キーを NULL に

2-3. マイグレーション順序

外部キーで参照される側（genres）が先に作られる必要がある。Laravelはファイル名の日時プレフィックスで実行順を決めるため、genres のファイルが books より古い日時になっていることを確認する。

2-4. シーダーの役割

GenreSeeder でジャンルマスタ（小説・ビジネス・技術書 等）を投入。firstOrCreate を使うことで、シーダーの再実行でも重複を避けられる。
