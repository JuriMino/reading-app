JSの解説（ステータスバッジ化＋即時保存）

やること: ステータス表示をプルダウン単体からバッジ風の色付き表示に変更し、一覧画面では選択を変えた瞬間に非同期でDB保存＆色も即時更新する。

1. 実装全体像

ユーザー体験として目指すのは「一覧のステータスプルダウンを変えると、リロードなしで保存され、色もその場で変わる」こと。これを実現するには：

  1. 各ステータスに対応する色（CSSクラス）を Enum 側に定義
  2. ビュー（一覧・詳細）でそのクラスを出力 → 見た目をバッジ化
  3. JS で <select> の change イベントを監視 → fetch でPATCH送信
  4. サーバーが新しい badgeClass を返す → JS でクラスを付け替えて色変更

2. 前提：ステータス表示のバッジ化

2-1. Enum に badgeClass() メソッドを追加

app/Enums/BookStatus.php

public function badgeClass(): string
{
    return match($this) {
        self::Unread  => 'bg-red-100 text-red-700 border-red-200',
        self::Reading => 'bg-blue-100 text-blue-700 border-blue-200',
        self::Done    => 'bg-green-100 text-green-700 border-green-200',
    };
}
ビューに色分岐を書くとロジックが散らばる。Enumにメソッドとして持たせれば、一覧・詳細・API応答のどこからでも同じロジックを再利用できる。

2-2. 詳細画面：固定バッジ化

resources/views/books/show.blade.php のステータス部分を編集不可な固定表示に。

<dd>
    <span class="inline-flex items-center gap-1 px-3 py-1 rounded-full text-xs font-medium border {{ $book->status->badgeClass() }}">
        ● {{ $book->status->label() }}
    </span>
</dd>

2-3. 一覧画面：バッジ風プルダウン

<select> 自体に badgeClass() を適用し、見た目はバッジ・機能は編集可能なプルダウンに。

<select class="status-select {{ $book->status->badgeClass() }} border rounded-full px-3 py-1 text-xs font-medium focus:outline-none focus:ring-2 focus:ring-blue-500"
        data-book-id="{{ $book->id }}">
    @foreach ($statuses as $status)
        <option value="{{ $status->value }}" @selected($book->status === $status)>
            {{ $status->label() }}
        </option>
    @endforeach
</select>

3. サーバー側：レスポンスに色クラスを含める

BookController@updateStatus

public function updateStatus(Request $request, Book $book)
{
    $validated = $request->validate([
        'status' => ['required', Rule::enum(BookStatus::class)],
    ]);

    $book->update($validated);

    return response()->json([
        'status'     => $book->status->value,
        'badgeClass' => $book->status->badgeClass(),
    ]);
}

色の切り替えロジックをJS側に持たせるとEnumと二重管理になる。「現在のステータスに対応する色クラス文字列」をサーバーに返してもらえば、JSは受け取ったクラスを付けるだけで済む。

4. JSの実装

resources/js/app.js

document.addEventListener('DOMContentLoaded', () => {
    const tokenMeta = document.querySelector('meta[name="csrf-token"]');
    if (!tokenMeta) return;
    const token = tokenMeta.content;

    // 付け替え対象の色クラス（全ステータスぶんを列挙、各要素は単一クラス名）
    const COLOR_CLASSES = [
        'bg-red-100', 'text-red-700', 'border-red-200',
        'bg-blue-100', 'text-blue-700', 'border-blue-200',
        'bg-green-100', 'text-green-700', 'border-green-200',
    ];

    document.querySelectorAll('.status-select').forEach((select) => {
        select.addEventListener('change', async () => {
            const bookId = select.dataset.bookId;
            const status = select.value;

            try {
                const res = await fetch(`/books/${bookId}/status`, {
                    method: 'PATCH',
                    headers: {
                        'Content-Type': 'application/json',
                        'Accept': 'application/json',
                        'X-CSRF-TOKEN': token,
                    },
                    body: JSON.stringify({ status }),
                });

                if (!res.ok) throw new Error('保存に失敗しました');

                const data = await res.json();

                // 古い色クラスを全部外して、新しい色クラスを付与
                select.classList.remove(...COLOR_CLASSES);
                select.classList.add(...data.badgeClass.split(' '));
            } catch (err) {
                alert(err.message);
            }
        });
    });
});

5. 各パーツのポイント解説

5-1. CSRFトークンを meta タグから取る

Laravel は POST/PUT/PATCH/DELETE に CSRFトークンを要求する。フォーム送信なら @csrf が自動でhiddenを入れてくれるが、JS経由の fetch では自分でヘッダに付ける必要がある。共通レイアウトに以下が入っている前提：

<meta name="csrf-token" content="{{ csrf_token() }}">

JSでは document.querySelector('meta[name="csrf-token"]').content で取り出し、fetchの X-CSRF-TOKEN ヘッダに入れる。

5-2. dataset で data-* 属性を読む

HTMLJS
data-book-id="5"element.dataset.bookId
data-user-name="taro"element.dataset.userName

ハイフン区切りが自動でキャメルケースに変換される。

5-3. classList でクラス操作

element.classList.add('foo', 'bar');      // 追加
element.classList.remove('foo', 'bar');   // 削除
element.classList.toggle('foo');          // オンオフ切替
element.classList.contains('foo');        // 有無チェック

引数は1要素1クラス名。スペース区切りの文字列を1つ渡すとエラーになる（後述）。

5-4. スプレッド構文 ...array

classList.remove(...COLOR_CLASSES) は配列の各要素を個別の引数として展開する書き方。以下と等価：

classList.remove('bg-red-100', 'text-red-700', 'border-red-200', ...);

5-5. Vite dev server

開発中は Vite を起動しておく必要がある（@vite ディレクティブが Vite から JS・CSS を読み込むため）。別ターミナルで：

sail npm run dev

6. 発生したエラー・原因・対応

6-1. プルダウン変更しても何も起きない

原因: resources/views/layouts/app.blade.php の @vite ディレクティブが recources/js/app.js（タイポ）になっていて、JS が読み込まれていなかった。

対応: resources/js/app.js に修正。ビューのソースを表示して<script>タグの src を確認するのが早い。

6-2. Cannot read properties of undefined (reading 'bookId')

原因: プロパティ名のタイポ。select.database.bookId （正：dataset）、select.ariaValueMax（正：value）。

対応:

  • database → dataset：data-*属性を取るプロパティ名
  • ariaValueMax → value：<select>や<input>の値を取る標準プロパティ
教訓: エディタの補完に頼っていると、存在はするが全く別物のプロパティ（ariaValueMax等）を選んでしまうことがある。dataset・valueは標準中の標準なので暗記しておく。

6-3. Failed to execute 'remove' on 'DOMTokenList': The token provided contains HTML space characters

原因: COLOR_CLASSES 配列の各要素にスペース区切りで複数クラスを入れていた。 
// NG
const COLOR_CLASSES = [
    'bg-red-100 text-red-700 border-red-200',   // ← 1要素にスペース入り
    ...
];
classList.remove は1引数=1クラス名が必須で、スペース入りの文字列は受け付けない。

対応: 各クラスを配列の別要素に分割。

// OK
const COLOR_CLASSES = [
    'bg-red-100', 'text-red-700', 'border-red-200',
    'bg-blue-100', 'text-blue-700', 'border-blue-200',
    'bg-green-100', 'text-green-700', 'border-green-200',
];

サーバーから受け取る data.badgeClass はスペース区切り文字列なので、.split(' ') で分解してから add に渡す。

7. 動作確認

  1. 一覧画面でステータスプルダウンを変更 → ページ遷移なし
  2. 色もその瞬間に変わる（未読=赤、読書中=青、読了=緑）
  3. F5でリロードしても変更が反映されている → DB保存成功
  4. DevTools の Network タブで PATCH /books/{id}/status が 200 を返していることを確認
