reading-record 開発記録

日付: 2026-04-17

概要: ビュー作成を実施。共通レイアウト、一覧、詳細、作成・編集フォーム、フォーム部品（パーシャル）までを構築。CSSはTailwind CSS v4 を使用。書籍登録・更新・削除が動作する状態に到達。

1. 今日の作業手順

1-1. Tailwind CSS の準備

プロジェクトに既にTailwind v4 がセットアップ済みだったため、別ターミナルで Vite 開発サーバーを起動:

sail npm install
sail npm run dev

1-2. 共通レイアウト

resources/views/layouts/app.blade.php を作成。ヘッダ・メイン部・CSRFメタタグ・Viteアセット読み込みを含む。子ビューで @yield('title') と @yield('content') を差し込む構成。

1-3. 一覧画面

books/index.blade.php を作成。テーブル表示、各行にタイトルへのリンク、ステータスのプルダウン（即保存は後日）、編集・削除ボタン。

1-4. 詳細画面

books/show.blade.php を作成。書籍情報、要約、感想をセクション分けして表示。ステータスは詳細でもプルダウン。

1-5. 作成・編集フォーム

入力項目が共通なのでパーシャル（_form.blade.php）に切り出し、create.blade.php と edit.blade.php から @include('books._form', [...]) で読み込む構成に。

2. 発見したエラーと原因・対応

2-1. Route [book.index] not defined

原因: routes/web.php のリダイレクトで route('book.index')（単数形）と記述。ルート名は複数形 books.index で定義していた。

対応: 'book.index' → 'books.index' に修正。

2-2. Route [books.index] not defined

原因: ルートグループ名の指定 name('books') の末尾にドットが無かった。結果、name('books') + name('index') が連結されて booksindex というルート名になっていた。

対応: name('books.')（末尾ドット付き）に修正。
ポイント: グルーピング時のプレフィックス連結には、ドットは自動で付与されないので明示する必要がある。

2-3. Undefined constant "books"

原因: Bladeで {{ route(books.store) }} のように、ルート名をクォートせず記述。PHPが books を定数として解釈しようとして失敗。

対応: {{ route('books.store') }} とクォートで囲む。文字列リテラルは必ずクォートが必要。

2-4. @include のデータ渡しで配列キーを変数として書いてしまった

原因: @include('books._form', [$book => null]) と記述。配列キーが変数になっている。

対応: ['book' => null] と文字列でキー指定。

2-5. Attempt to read property "genre_id" on null

原因: _form.blade.php で $book->genre_id と書いていた。create時は $book が null のためエラー。

対応: Null安全演算子 $book?->genre_id を使う。

2-6. Undefined variable $genre_id

原因: @foreach ($genres as $genre) の中で、選択比較に $genre_id と書いてしまった。正しくはループ変数の $genre->id。

対応: @selected(old('genre_id', $book?->genre_id) == $genre->id) に修正。比較は ==（緩い等価）にして型違いを吸収。

2-7. Field 'author' doesn't have a default value (MySQL)

原因: Book モデルの $fillable に typo があり、'autohr' と記述されていた。結果、リクエストに author が含まれていても Mass Assignment で除外されてSQLに含まれず、NOT NULL制約でエラー。

対応: 'autohr' → 'author' に修正。
教訓: $fillable のtypoは、バリデーション通過後でも黙って無視されるため気づきにくい。モデル編集後は実際に登録まで動作確認。

2-8. Undefined variable $statuses

原因: BookController の index と show で、ビューに渡すキーが 'status'（単数）になっていた。ビュー側は $statuses（複数）を期待。

対応: コントローラ側を 'statuses' => BookStatus::cases() に統一。

2-9. POST method is not supported for route books/{id}

原因: 編集フォームに @method('PUT') が無く、POSTで送信されていた。ルートは PUT/DELETE のみ受付。

対応: edit.blade.php のフォーム内に @method('PUT') を追加。修正後も症状が残ったのはビューキャッシュが原因で、sail artisan view:clear + ブラウザのスーパーリロードで解決。

Tip: HTMLフォームはGET/POSTしか送れない。PUT/PATCH/DELETEは @method('PUT') 等で hidden input の _method を出力してLaravelに解釈させる。

2-10. update メソッド内の静的呼び出し誤り

原因: Book::update($request->validated()) と記述。静的呼び出しだと対象レコードが曖昧になる。

対応: $book->update($request->validated()) に修正。ルートモデルバインディングで取得したインスタンスのメソッドを呼ぶ。

2-11. 詳細画面の削除フォームの route() で引数不足

原因: route('books.destroy') と書いていた（$book 引数なし）。

対応: route('books.destroy', $book) に修正。

2-12. 詳細画面の編集・削除ボタンのCSS崩れ

原因: 複合的な崩れ。
  • 外側コンテナが fex gap-2（Tailwindに存在しないクラス）で flex が効かず縦並びに
  • 編集ボタンが bg-gray-600 + text-gray-700 でコントラスト不足で読みにくい
  • タイトルが長いとボタン側が Flexbox のデフォルト挙動で縮められ「編／集」のように縦書きに

対応:

  • fex → flex に修正
  • ボタン色を bg-gray-100 + text-gray-700 に変更
  • ボタンに whitespace-nowrap、ボタン群に shrink-0、タイトル側に min-w-0 flex-1 と break-words を付与

3. 今日得た知見（まとめ）

3-1. Blade の共通パターン

書き方意味
@extends('layouts.app')共通レイアウトを継承
@section('xxx') ... @endsectionレイアウトの @yield('xxx') に差し込む
@include('books._form', [...])パーシャル読み込み。先頭 _ はパーシャル慣習
@csrfCSRFトークンhidden input
@method('PUT')PUT/DELETE偽装用hidden input
@selected($条件)条件trueなら selected 属性を出力
old('key', $default)バリデーション失敗後のフォーム再表示用
$book?->titleNull安全演算子。nullでもエラーにならない

3-2. Tailwind の色の濃淡

  • 50 = 一番薄い（背景のうっすら色）
  • 100〜300 = 薄め（ボタン背景など）
  • 500 = 中間
  • 600〜800 = 濃い（文字色など）
  • 900 = 一番濃い

背景と文字は濃淡を逆に組み合わせる（例: bg-gray-100 + text-gray-700）のが視認性の基本。

3-3. Flexbox の挙動とボタン潰れ

  • Flex子要素はデフォルトで自動的に縮められる
  • 縮めたくない要素は shrink-0
  • テキストを改行させたくないなら whitespace-nowrap
  • 長文側は min-w-0 flex-1 + break-words で適切に折り返す

3-4. typo と関連エラーのパターン

  • ルート名ドット抜け: name('books') → ルート名が連結されて意図と違うものに
  • $fillable typo: リクエストに値があっても Mass Assignment で黙って除外される → SQLエラー
  • ルート名クォート忘れ: 「Undefined constant」エラー
  • 変数名の単複混在: コントローラとビューでキー名が不一致
