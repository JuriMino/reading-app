ソート機能の実装

やること: 一覧画面にソート用のセレクトボックスを置き、登録日／タイトル／著者／読了日 × 昇順／降順で並び替えできるようにする。検索条件との併用も可能に。

1. 設計方針

  • UI：セレクトボックス1つで「登録日が新しい順」「タイトル昇順」などを並べる
  • 変更時挙動：プルダウンを変えた瞬間にフォームsubmit（onchange="this.form.submit()"）
  • デフォルト：登録日が新しい順（created_at DESC）
  • セキュリティ：ホワイトリスト方式（許可したソート種別のみ受け付ける）
  • 検索との組み合わせ：検索フォーム内にソートセレクトも置き、同じGETパラメータとして送る

2. コード

2-1. BookController@index にソート処理を追加

検索処理の後、$books = ... の直前に挿入。

// 許可するソートオプション（キー: 表示用識別子、値: [カラム, 昇降]）
$sortOptions = [
    'created_at_desc'  => ['created_at', 'desc'],
    'created_at_asc'   => ['created_at', 'asc'],
    'title_asc'        => ['title', 'asc'],
    'title_desc'       => ['title', 'desc'],
    'author_asc'       => ['author', 'asc'],
    'author_desc'      => ['author', 'desc'],
    'finished_at_desc' => ['finished_at', 'desc'],
    'finished_at_asc'  => ['finished_at', 'asc'],
];

// リクエストから受け取り、許可リストになければデフォルト
$sort = $request->input('sort', 'created_at_desc');
[$column, $direction] = $sortOptions[$sort] ?? $sortOptions['created_at_desc'];

$books = $query->orderBy($column, $direction)->get();

return view(...) の $filters には sort も含める：

'filters' => $request->only(['keyword', 'genre_id', 'status', 'sort']),

2-2. 検索フォーム内にソートセレクトを追加

index.blade.php の検索フォーム（既存の <div class="grid grid-cols-12 ...>）の下に1行追加。

<form action="{{ route('books.index') }}" method="get" class="...">
    {{-- 既存の検索行 --}}
    <div class="grid grid-cols-12 gap-3 items-end">...</div>

    {{-- ソート行を追加 --}}
    <div class="mt-3 flex items-center justify-end gap-2">
        <label class="text-xs text-gray-500">並び替え</label>
        <select name="sort" onchange="this.form.submit()"
                class="border border-gray-300 rounded-md px-3 py-2 text-sm">
            <option value="created_at_desc"  @selected(($filters['sort'] ?? 'created_at_desc') === 'created_at_desc')>登録日が新しい順</option>
            <option value="created_at_asc"   @selected(($filters['sort'] ?? '') === 'created_at_asc')>登録日が古い順</option>
            <option value="title_asc"        @selected(($filters['sort'] ?? '') === 'title_asc')>タイトル昇順</option>
            <option value="title_desc"       @selected(($filters['sort'] ?? '') === 'title_desc')>タイトル降順</option>
            <option value="author_asc"       @selected(($filters['sort'] ?? '') === 'author_asc')>著者昇順</option>
            <option value="author_desc"      @selected(($filters['sort'] ?? '') === 'author_desc')>著者降順</option>
            <option value="finished_at_desc" @selected(($filters['sort'] ?? '') === 'finished_at_desc')>読了日が新しい順</option>
            <option value="finished_at_asc"  @selected(($filters['sort'] ?? '') === 'finished_at_asc')>読了日が古い順</option>
        </select>
    </div>
</form>

3. ポイント解説

3-1. ホワイトリスト方式の重要性

仮にリクエストを無検証で orderBy($request->sort_column, $request->sort_direction) のように使うと、ユーザーがURLを書き換えて ?sort_column=password のようなおかしなカラム名を渡せてしまう。最悪の場合、DB構造を探るきっかけになる。

今回のように「このキーのとき、このカラムをこの向きで」という対応表をサーバー側で固定すれば、想定外の値はすべて無視され安全。

3-2. onchange="this.form.submit()" のUX

プルダウン変更時に自動でフォームsubmitするインラインJS。ユーザーは「検索」ボタンを押さずに即ソートできる。
検索テキストやステータスは明示的に「検索」ボタンで確定したいが、並び順は「変えたら即反映」のほうが直感的というケース。

3-3. 配列分割代入

[$column, $direction] = ['created_at', 'desc'];
// $column === 'created_at', $direction === 'desc'

PHP 7.1+ で利用可能。連想配列に対しても ['a' => $x, 'b' => $y] = $array; の形で使える。

3-4. ?? 演算子によるフォールバック

$sortOptions[$sort] ?? $sortOptions['created_at_desc']

左辺が null（またはキーが存在しない）のときに右辺を返す。「許可外のソートキーが来たらデフォルトに戻す」という保険。

3-5. デフォルト状態でもデフォルト選択肢がハイライトされる仕組み

初回アクセス時は $filters['sort'] が存在しない。そのままだと「登録日が新しい順」が選択状態にならない。
@selected(($filters['sort'] ?? 'created_at_desc') === 'created_at_desc') の ?? 'created_at_desc' がその救済。「未指定ならデフォルト値として扱う」ことで選択状態を復元している。

4. 検索機能との組み合わせ

ソートを検索フォームの内側に置いたことで、以下の挙動が自然に得られる：

  • 検索条件とソートが1つのGET URLに乗る（?keyword=Laravel&sort=title_asc）
  • 検索中にソートを変えても検索条件は維持される
  • ブックマーク時も並び順込みで保存できる
  • 「クリア」ボタンで検索条件とソート両方が初期化される

5. 動作確認

  1. ソートプルダウンを変更 → 自動で画面再読み込みされ並び替わる
  2. URLが ?sort=title_asc のように更新される
  3. 検索条件を入れたままソート変更 → 両方適用される
  4. 「クリア」で初期状態（登録日が新しい順）に戻る

6. 補足：表示との整合性

デフォルトが「登録日が新しい順」なのに、一覧テーブルに登録日列がないと利用者にとって不自然。
併せて index.blade.php のテーブルに 登録日 列を追加した（詳細はまとめの UI 調整を参照）。
